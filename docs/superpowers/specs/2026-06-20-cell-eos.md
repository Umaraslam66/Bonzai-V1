# Design spec ‚Äî cell-EOS (`<cell_end>` self-termination)

- **Date:** 2026-06-20
- **Branch:** `phase-2-cell-eos` (off `phase-2-bakeoff-2backbone`; this is a prerequisite to the scored bake-off ‚Äî the model must self-terminate before generation cost is bounded). Redirect to `main` if preferred.
- **Status:** DESIGN ‚Äî four review corrections folded in 2026-06-20 (terminator family / keep-260 / shape-first Tooth-1 / `(‚Ä¶,510,260)` Tooth-2). HALT for confirmation. No implementation, no cache rebuild, no scored run, no merge.

## 0. Problem (one paragraph)

The micro-AR generator has no stop signal: `generate_cell_tokens` runs `for _ in range(max_new)` with no `break`, and the training data carries no cell terminator (`<cell_end>` id 260 is `on_disk:false`, excluded from the vocab, and absent from every shard ‚Äî confirmed 0/23 real cells). So every generated cell runs to the `max_new` cap (the 89M diagnostic emitted exactly 2048/2048 tokens on 64/64 cells), which makes the held-out realism eval cost 68‚Äì118% of the grant instead of ~3‚Äì5%. This spec teaches the model to emit `<cell_end>` at cell end and stop there.

## 1. Locked decisions (do NOT re-litigate ‚Äî from prior investigation)

1. **Reuse id 260**, flip `<cell_end>` to `on_disk:true`. No new id ‚Üí `n_subf_vocab` stays 1508 ‚Üí head/embedding width unchanged ‚Üí 53M param-match untouched.
2. **Inject at `build_shards_in_memory`** (cached, derivation-versioned), with empty-guard `(*toks, 260) if toks else ()` ‚Äî NOT the in-code flatten append (post-cache; a 260-run and non-260-run would collide on the cache key ‚Äî unacceptable for the locked 6-run matrix).
3. **No `max_len` bump** ‚Äî the 260 counts inside the existing `DEFAULT_MAX_CELL_TOKENS=13312` budget. Measured: **0** train cells sit at body-length exactly 13,312, so this drops **zero** currently-kept cells (drop-not-clamp confirmed; 44 cells >13,312 already dropped, max 19,372).
4. **Bump `SHARD_CACHE_DERIVATION_VERSION` 1‚Üí2; rebuild caches** (fail-closed forces it, so no 260/non-260 mix possible).
5. **Generator: `break` on 260** in `generate.py`.

## 2. Verified facts the design rests on (citations)

- `load_sub_f_vocab()` (`src/cfm/data/sub_f/vocab.py:168-182`) concatenates five loaders ‚Äî semantic / unknown / encoding_primitive / structural / boundary_reference ‚Äî and **none read the `dataloader_sentinels` block**. So flipping `on_disk:true` in the YAML is **inert by itself**; a loader must pull 260 in (the fix adds a **sixth** loader `_load_cell_end_slot`, family `terminator` ‚Äî see 3.A.2).
- `n_subf_vocab = max(vocab_tag_to_id().values()) + 1 = 1508` (max on-disk id 1507). 260 < 1507 ‚Üí adding it does NOT change the max ‚Üí head `nn.Linear(d_model, n_subf_vocab)` (`scaffold_backbone.py:88`) and embedding (`:64`) widths unchanged ‚Üí param-match holds. `CONDITIONING_ID_BASE` (`conditioning.py:31`, same `max(...)+1`) also unchanged at 1508.
- The loss mask is **seq_len-relative** (`scaffold_backbone.py:156-162`): `target=ids[:,1:]`, mask `[:pl-1]` (prefix) and `[seq_len-1:]` (padding). Appending 260 (bumping `seq_len`) makes the `(last-510 ‚Üí 260)` transition fall in the supervised window automatically, and 260‚Üípad masked ‚Äî **no mask-code change needed** (Tooth 3 proves this).
- Both KS sides split via `split_cell_into_features` (`seam_decodability.py:67-85`), which drops a trailing 260 (after the last 510, `cur=[]`). Real reference cells (no 260) and generated cells (with 260) yield identical per-feature values ‚Üí **no KS bias**; the locked floor (sha `95abb88`) needs no re-extraction.

### B. build_shards ‚Äî inject 260 at non-empty cell-end (empty-guard)

## 4. TEETH (all must pass before trust ‚Äî before any scored run)

> Ordering: A‚ÜíB/C land + CPU teeth (2,3,4) green ‚Üí cache rebuild ‚Üí smoke-train tooth (1). Tooth 1 needs a FRESH train on the v2 cache; the existing diagnostic checkpoints were trained WITHOUT 260 and CANNOT be reused.

**Tooth 1 ‚Äî gen EMITS 260 and UNPINS from the cap (GPU smoke).** After a short smoke train on the v2 (260) cache, generate N cells under matched conditioning. **PASS is SHAPE-FIRST, NOT a 95% emit-rate** ‚Äî a sub-1-GPU-h smoke can underlearn a once-per-cell target, so a 95% gate risks a FALSE RED that is *undertraining, not broken wiring*. PASS requires BOTH:
- **(a) length distribution clearly UNPINNED from the cap** ‚Äî p99 well below `max_new`, median in the low-hundreds-to-~600 band (the real ~600 region), NOT a spike at `max_new`; AND
- **(b) emit-rate materially > 0** ‚Äî a decisive contrast against the pre-fix baseline (0% emit / 100% of cells at the cap).

Together these prove the wiring learned to stop. **Reserve ‚â•95% emit for the SCORED runs** (a fully-trained model), not this smoke. **Smoke budget: sub-1-GPU-h `fast_dev`** (NOT a full run); the existing diagnostic checkpoints were trained WITHOUT 260 and CANNOT be reused.

**Tooth 2 ‚Äî rebuilt cache: byte-deterministic, 260 only at non-empty cell-end, ends `(‚Ä¶,510,260)`, 0 empties carry it.**
- `tests/training/test_shard_cache.py:91` `test_city_cache_bytes_are_deterministic` ‚Äî re-run on v2, GREEN (determinism preserved).
- NEW data-integrity check over the rebuilt cache `tokens`: for every cell ‚Äî `tokens[-1] == 260` **iff** non-empty; `tokens.count(260) == 1` for non-empty (260 never interior: `260 not in tokens[:-1]`); empty cells `tokens == ()` (no 260); **AND every non-empty cell ends exactly `(‚Ä¶, 510, 260)`** ‚Äî i.e. `tokens[-2] == 510` (the last feature's `<feature_end>` immediately precedes the cell terminator). This `(510, 260)` assertion proves the real cell-ends match the `510 ‚Üí 260` transition that Tooth 3 supervises on a **synthetic** fixture ‚Äî so Tooth 3 is testing a REPRESENTATIVE cell-end, not an assumed one. Run as a fixture-level unit test on `build_shards_in_memory` output AND a spot-scan of the real rebuilt cache.

**Tooth 3 ‚Äî loss supervises the 260 target.** Unit test: construct a `CellExample` whose body ends `‚Ä¶, 510, 260`; call `training_loss`; assert (a) the next-token target at the final-510 position **== 260** and is **not** `_IGNORE` (it contributes to CE; `n_supervised_positions` counts it); (b) the 260's own next-target (first pad) **is** masked. Proves the model is trained to emit 260 with no mask-code change.

**Tooth 4 ‚Äî golden fixture regenerated for derivation v2 (red-before / green-after).** Before re-goldening: with `SHARD_CACHE_DERIVATION_VERSION="2"` + the build_shards change, `test_golden_fixture_pins_versions_to_the_cache_bytes` is **RED** (both `derivation` "1"‚âÝ"2" and `cells_sha256` mismatch). After updating the asserted `derivation‚Üí"2"` and the new `cells_sha256`: **GREEN**. The red phase is the proof the bump+format change were not silent. *Capture both phases in the PR/report.*
