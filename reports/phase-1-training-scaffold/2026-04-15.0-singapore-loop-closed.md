## Sub-project summary (training-scaffold, 2026-06-02)

**What this proves.** The first end-to-end loop on the project is CLOSED on real
Leonardo 4xA100 hardware: build training shards (validated 362 tiles minus the 132
holdout, by ID from the frozen manifest) → `CellDataModule` fail-closed holdout audit
at `setup()` → train a toy decoder-only micro-generator (4-GPU DDP, bf16) →
checkpoint + resume → generate cells → decode via the SEALED sub-F decoder → one real
per-cell eval number on Singapore. Job `43947093`, 500 steps, 8m43s.

**Interpreting the metrics.**
- `decodability_rate=0.922` — 1685 of 1827 generated feature blocks decode through the
  sealed decoder without error. The generate→decode seam works on model output.
- `ogc_valid_rate=0.983` — of the decoded, non-bref-collapse geometries, 98.3% are
  OGC-valid. Validity excludes the v1 outbound-bref placeholder collapse by sub-G's
  construction-identity predicate (structural exclusion), which is reported separately.
- `bref_collapse_rate=0.028` — the known v1 limitation, REPORTED-not-gated.
- `right_angle_rate=0.0` — **expected for a 500-step, value-agnostic-conditioning toy**:
  rectilinearity is a *learned* property the bake-off pursues, not a slice deliverable.
  The metric now also emits `n_polygons`/`n_corners` (added post-run) so a future 0.0 is
  disambiguated (no polygons vs polygons without right angles); for THIS run the 0.0 is
  the un-enriched value. This is a reported-not-gated quality signal, not a gate.

**DDP guarantees (non-vacuous; `world_size==4` asserted, 4 real ranks).**
- All-ranks holdout audit-halt-before-batch-0: **PASS** — a planted leak halts
  `setup()` on all 4 ranks (`ranks_halted_before_batch0=4/4`), zero training steps.
- Bit-identical 4→4 resume: **functionally identical** — after fixing nondeterministic
  weight init (seed before model construction), an uninterrupted reference vs a
  checkpoint-and-resume run differ only at float32 epsilon (`max|diff|=1.19e-07`; 5/28
  tensors, LayerNorm biases + embedding tables). Exact bit-identity is unachievable
  under DDP NCCL all-reduce reduction-order nondeterminism; functional identity at
  `atol=1e-4` is the correct bar, evidenced by the per-tensor magnitudes.

**Comparability lock.** torch 2.5.1+cu121 / lightning 2.6.5 / pydantic 2.13.4, asserted
at every GPU entrypoint (`assert_training_env_locked`). tensorboard is NOT in the lock
